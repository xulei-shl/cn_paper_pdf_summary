# 开发变更记录
- **日期**: 2026-04-30
- **对应设计文档**: `docs/design/telegram结束日志与正文推送解耦_20260430.md`

## 1. 变更摘要
- 为 `/process` 新增 `request_source` 字段，用于区分 Telegram Bot 与普通 API 场景
- Telegram Bot 调用 API 时显式传入 `request_source=telegram`
- Telegram 命令场景固定返回结束日志，不再额外触发 `PDF_SUMMARY_PUSH_TELEGRAM_LOG` 对应的主动成功日志
- `PDF_SUMMARY_PUSH_TELEGRAM_RESULT` 继续统一控制最终摘要正文推送
- 抽离 Telegram 结束日志格式化模块，并补充 Node 测试

## 2. 文件清单
- `api.py`: 修改，请求模型新增 `request_source`
- `utils/api_queue.py`: 修改，透传请求来源并按来源控制成功日志推送
- `utils/notifier.py`: 修改，成功通知支持按入口抑制 Telegram 成功日志
- `tests/test_notifier.py`: 修改，新增 Telegram 来源下的通知测试
- `telegram-bot/index.ts`: 修改，调用 API 时声明 `telegram` 来源并复用格式化模块
- `telegram-bot/response-formatter.ts`: 新增，封装 Telegram 结束日志文案
- `telegram-bot/response-formatter.test.ts`: 新增，覆盖成功/失败结束日志
- `telegram-bot/package.json`: 修改，纳入新增测试文件
