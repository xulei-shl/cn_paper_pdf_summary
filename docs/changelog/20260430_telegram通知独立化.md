# 开发变更记录
- **日期**: 2026-04-30
- **对应设计文档**: `docs/design/telegram通知独立化_20260430.md`

## 1. 变更摘要
- 新增本地通知分发器，移除主项目统一推送依赖
- Telegram `/papers` 命令去掉 `@ID` 语义，改为纯标题直连
- 成功日志、成功结果、错误通知按新配置分别控制
- WeChat 是否发送改为仅由 `.env` 与 `--wechat` 控制，`config.yaml` 仅保留技术参数
- `api.py` 的请求/响应模型与接口描述已同步到最新语义，避免 `id` / `push_wechat` / `notify` 产生歧义
- Blinko 配置已统一到项目根目录 `.env`，不再默认读取子模块独立 `.env`
- 已删除数据库批量模式及相关冗余配置，主流程只保留直连单篇处理

- `summary-update/blinko-api/src/blinko_client/base.py`: 统一 Blinko 配置读取入口
- `tests/test_blinko_config.py`: 新增 Blinko 配置加载测试
- `config/config.yaml`: 删除数据库批量模式相关配置
- `utils/database.py`: 删除未使用的数据库辅助模块
