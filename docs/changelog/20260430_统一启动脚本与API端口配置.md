# 开发变更记录
- **日期**: 2026-04-30
- **对应设计文档**: `docs/design/统一启动脚本与API端口配置_20260430.md`

## 1. 变更摘要
- 新增统一启动脚本 `start_all.sh`，支持先启动 API，再等待健康检查通过后启动 Telegram Bot。
- 为 API 启动增加环境变量端口配置，避免 `8081` 写死在代码中。
- 为 Telegram Bot 增加 API 地址解析模块，优先使用显式 `TELEGRAM_API_URL`，否则按 `PDF_SUMMARY_API_PORT` 回退到本地地址。
- 补充 Python 与 TypeScript 侧的配置解析测试用例。

## 2. 文件清单
- `api.py`: 新增 API 监听地址与端口环境变量读取逻辑

- `telegram-bot/api-config.ts`: 新增 Bot 侧 API 地址解析模块
- `telegram-bot/index.ts`: 接入 API 地址解析逻辑
- `telegram-bot/api-config.test.ts`: 新增 API 地址解析测试
