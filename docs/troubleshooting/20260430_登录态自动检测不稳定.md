# 故障复盘报告
- **日期**: 2026-04-30
- **严重级别**: Medium

## 1. 问题现象
执行 `python summary-update/hiagent-rag-upload/upload_knowledge.py --login-only --headless=False` 后，用户已在浏览器中手动登录成功，但脚本仍停留在“未检测到登录状态，请在浏览器中手动完成登录...”阶段，无法继续导出登录状态。

## 2. 根本原因 (Root Cause)
`--login-only` 流程使用页面中的“导入文件”按钮作为登录成功判定条件。因为手动登录完成后，页面未必自动回到目标知识库页，或者目标按钮的加载时机和可见性不稳定，所以自动检测条件并不可靠。因为检测条件不可靠，所以脚本即使在真实登录成功后也可能一直阻塞，无法进入导出流程。

## 3. 解决方案
将 `--login-only` 流程改为终端人工确认模式：浏览器打开后由用户手动登录，登录完成后在命令行按回车确认，脚本执行一次页面刷新后导出登录状态；如用户输入 `q`，则主动取消本次导出。上传主流程保持不变。

## 4. 预防措施
后续涉及第三方站点登录态判断时，优先避免将单一页面元素作为唯一成功信号。对于人工参与的登录流程，优先提供显式终端确认机制，并保留必要的页面刷新或二次检查步骤，降低页面跳转和异步渲染导致的误判风险。
